Privacy Policy for Notepad App

Last Updated: August 26, 2025

JoshTechApps ("we," "us," or "our") is committed to protecting your privacy. This Privacy Policy outlines how the Notepad app collects, uses, stores, shares, and protects your personal information. By using the app, you consent to these practices. For inquiries, contact us at contactjoshtech@gmail.com.

1. Information We Collect

a. User-Generated Data

  • Notes: You can create and manage notes stored locally in the app’s database, including:
    • Title and content
    • Note type (text, checklist, to-do)
    • Creation and update timestamps
    • Reminder settings (time, days, weekly repeat)
    • Theme preferences
    • Passwords and security questions/answers (optional, for protected notes)
    • Status flags (archived, trashed, notified, selected)
  • Calendar Interactions: The app accesses note creation and reminder dates for calendar view display.
  • User Interactions: Actions like creating, editing, sharing, archiving, trashing, or setting reminders are recorded locally for functionality.

b. User Preferences

Stored locally in the app’s preferences:

  • Theme mode (light, dark, system default)
  • View type (grid or list)
  • Font size for note display
  • Sort order (modified time, created time, alphabetical)
  • Language selection (English, German, French, Spanish, Portuguese, Russian, Swahili, Arabic, Bengali, Hindi, Chinese, Filipino, Italian, Luganda)
  • Auto-save toggle
  • Notification sound (default, custom ringtones, or none)
  • App password (optional)
  • Fingerprint authentication setting
  • Remember me setting and timestamp (24-hour authentication bypass)
  • Notification Sounds: Stores URI of selected audio files for custom ringtones.

c. Authentication Data

  • Google Sign-In: For optional backups, we collect:
    • Google account email and display name (via Firebase Authentication)
  • Passwords: App and note-specific passwords, stored locally in plain text.
  • Security Questions/Answers: For password recovery, stored locally.
  • Fingerprint Authentication: Optional biometric data, managed by your device.

d. Device and Permission Data

  • Permissions:
    • POST_NOTIFICATIONS: For reminder notifications.
    • INTERNET: For Google Sign-In, Drive backups, ads, analytics, and Crashlytics.
    • SCHEDULE_EXACT_ALARM: For precise reminders.
    • READ_MEDIA_AUDIO or READ_EXTERNAL_STORAGE (pre-Android 11): For custom notification sounds.
    • RECEIVE_BOOT_COMPLETED: To reschedule reminders after reboot.
    • REQUEST_IGNORE_BATTERY_OPTIMIZATIONS: For reliable reminders.
  • Device Data: Custom ringtone URIs and metadata (e.g., file names).

e. Analytics and Crash Reporting

  • Firebase Analytics: Collects anonymized data (screen views, button clicks, session duration) to improve user experience.
  • Firebase Crashlytics: Collects crash logs, stack traces, and diagnostics (device info, battery usage) to enhance app stability.

f. Advertising Data

  • Google AdMob: Collects device identifiers (Advertising ID), IP address, and usage data for personalized ads.

2. How We Use Your Information

  • Notes and Calendar: Create, edit, display, and organize notes in a calendar view.
  • Reminders: Schedule notifications with note content using AlarmManager.
  • Settings: Personalize theme, language, font, view, sort order, auto-save, and sounds.
  • Authentication: Secure app/notes with passwords, questions, or biometrics.
  • Backups: Save/restore notes to Google Drive (user-initiated).
  • Notifications: Display reminders, bypassing DND for reliability.
  • Analytics and Crash Reporting: Improve app performance and stability via Firebase.
  • Ads: Deliver personalized/non-personalized ads via AdMob.
  • Support: Process feedback sent to contactjoshtech@gmail.com.
  • Language Support: Apply preferred language, including RTL for Arabic.

3. Data Storage

  • Local Storage: Notes, preferences, passwords, and security answers stored in a secure database. Note: Passwords stored in plain text; secure your device.
  • Cloud Storage: Backups stored in Google Drive’s appDataFolder; analytics/crash data on Firebase servers.
  • Retention:
    • Local data retained until deleted or app uninstalled.
    • Trashed notes retained until permanently deleted/restored.
    • Cloud backups persist until deleted in Google Drive.
    • Analytics/crash data retained per Google’s policies.
  • Logs: Debugging logs stored locally; crash logs sent to Firebase.

4. Data Sharing

  • Google Services:
    • Firebase Authentication: Manages Google Sign-In. See Google’s Privacy Policy.
    • Google Drive API: Stores encrypted backups.
    • Firebase Analytics/Crashlytics: Shares anonymized usage/crash data. See Google’s Privacy Policy.
    • Google AdMob: Serves ads. See AdMob’s Privacy Policy.
  • User-Initiated Sharing:
    • Note Sharing: Share notes via external apps (e.g., email).
    • App Sharing: Share app details or Play Store link.
    • Support: Feedback may include user data.
  • No Other Sharing: Data not shared except as described or with consent.

5. Data Security

  • Encryption: HTTPS for Google Drive/Firebase transfers.
  • Local Security: Device-protected storage; plain-text passwords pose a risk.
  • Biometrics: Fingerprint authentication via device system.
  • Notifications: Note content on lock screen, controlled by device settings.
  • DND Access: Bypasses Do Not Disturb for reminders.

6. User Controls

  • Notes: Create, edit, lock, archive, trash, restore, or delete.
  • Backups: Enable/disable or delete Google Drive backups.
  • Settings: Manage auto-save, sounds, passwords, biometrics, themes, views, fonts, sort orders, and languages.
  • Analytics/Crashlytics: Opt out via device settings or uninstallation.
  • Ads: Opt out of personalized ads via AdMob/device settings.
  • Notifications: Control visibility/sound via device settings.
  • Permissions: Grant/deny via system prompts.
  • Data Deletion: Delete local data via trash/uninstallation; cloud data via Google Drive. Request analytics deletion at contactjoshtech@gmail.com.

7. Permissions

  • POST_NOTIFICATIONS: Reminder notifications.
  • INTERNET: Sign-in, backups, ads, analytics, Crashlytics.
  • SCHEDULE_EXACT_ALARM: Precise reminders.
  • READ_MEDIA_AUDIO/READ_EXTERNAL_STORAGE: Custom sounds.
  • RECEIVE_BOOT_COMPLETED: Reschedule reminders post-reboot.
  • REQUEST_IGNORE_BATTERY_OPTIMIZATIONS: Reliable reminders.

8. Third-Party Services

  • Firebase Authentication: Google Sign-In. Google’s Privacy Policy.
  • Firebase Analytics: Usage data. Google’s Privacy Policy.
  • Firebase Crashlytics: Crash logs. Google’s Privacy Policy.
  • Google Drive API: Backups/restores.
  • Google AdMob: Ads. AdMob’s Privacy Policy.

9. Your Rights

  • Access: View/export notes, preferences, backups.
  • Delete: Delete local data (trash/uninstall); cloud data (Google Drive); analytics data (contact us).
  • Opt-Out: Disable personalized ads or analytics.
  • Privacy Requests: Email contactjoshtech@gmail.com for GDPR/CCPA rights.

10. Children’s Privacy

If the app is intended for users under 13 (or 16 in some regions), we comply with the Children’s Online Privacy Protection Act (COPPA) and GDPR-K by limiting data collection and requiring parental consent. Please contact us at contactjoshtech@gmail.com for details on parental controls or to provide consent.

11. Changes to This Policy

Updates will be posted in-app (Settings) and on our website, with Google Play Store notifications. Continued use implies acceptance.

12. Contact Us

Email contactjoshtech@gmail.com for questions or privacy rights.
